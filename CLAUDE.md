# CLAUDE.md — AI校長 モックアップ プロジェクトノート

## プロジェクト概要
AI校長アプリのクリッカブルモックアップ（HTML/CSS/JS）を作成するプロジェクト。
仕様書: `ai_kocho_spec.md`

## 技術方針
- **単一HTMLファイル** (`index.html`) で全画面を管理
- JS で画面の表示/非表示を切り替えるSPAスタイル
- 外部ライブラリなし（CDN含む）、純粋なHTML/CSS/JS
- レスポンシブ: 管理者・先生画面はデスクトップ、生徒画面はモバイル幅

## ユーザー区分
| 区分 | アクセス方法 | 主な画面 |
|------|-------------|---------|
| 管理者 | ログイン | 学校一覧(進捗ダッシュボード)、学校セットアップ、学校詳細 |
| 学校の先生 | ログイン | ダッシュボード(進捗バナー含む)、授業運営 |
| 校長/先生(インタビュー) | 専用URL | 事前インタビューフロー |
| 生徒 | 専用URL | 名前入力〜会話〜ポジショニング |

## ワークフロー(v3.0)
実施学校決定 → 管理者が学校セットアップ(基本情報+校長情報+Ver選択+テーマ+AI下書き生成+インタビューURL発行) → 校長先生へインタビュー依頼 → 回答反映でモデル稼働
- 1校=1ワークスペース(`admin-school-detail`)に全情報集約
- 進捗6ステップ: 基本情報 / テーマ設定 / 素材収集 / 下書き生成 / インタビュー送付 / モデル稼働
- 下書きは必ず作成(スキップ不可)。管理者と先生で同じ進捗が見える

## ダミーデータ(v3.5 で匿名化)
- 学校: A中学校(主, 旧:湯田)、B小学校(旧:大内)、C中学校(旧:平川)、D小学校(旧:白石)
- 校長: 例)山田 太郎(A中学校) — 人名には `例)` プレフィックスで例示であることを明示
- 生徒: 田中 花子、鈴木 一郎、佐藤 健太、高橋 美咲 等(こちらは架空としてそのまま)
- 授業テーマ(本題トーク): デフォルト「学校や地域、山口の魅力について話してみよう」(1つ固定。オプションで変更可)
- アイスブレイクテーマ: 「AI校長に質問してみよう」
- 方言: 山口弁(「〜じゃけぇ」「〜しちょる」)
- AI回答例の食べ物: 瓦そば(山口の郷土料理)
- ※山口の地域色(湯田温泉・七夕祭りなど)は地域情景としてそのまま保持。学校名・校長名のみ匿名化
- ※東北弁・ずんだ餅などの東北要素は使用しない

## 実装メモ
- 画面切替は `show(id)` 関数で `.screen.active` を付け替え
- デモナビは `#devnav`（右下固定）。ヘッダクリックで折りたたみ
- 授業運営のフェーズは **v3.2 で5段階** に拡張: `PHASES_V32 = [icebreak, summary-1, talk, summary-2, positioning]`
  - 各要素に `type: 'talk'|'summary'|'positioning'` を持ち、`renderCenterViewForPhase()` が中央ビューを自動切替
  - `advancePhase()` で順送り、最終フェーズ後は confirm → `teacher-review` へ
  - サマリーフェーズは `#classroom-view-summary`(編集可能本文 + AIへの指示チャット + 保存/投影)を表示
  - v3.1 のフェーズ遷移モーダル(`#phase-end-modal`)は v3.2 で廃止
- 生徒画面は PC幅レイアウト（v2より変更）。トップバー中央寄せ + フル幅会話 + フローティングメモ
- 7-5-C にフェーズ名は出さず `.student-theme-bar` でトーク内容のみ表示
- インタビューの本人/代理Ver切替は `.ver-honnin` / `.ver-dairi` クラス + `.ver-badge` をJSの `selectVer()` で表示制御
- サイドバーは管理者用・先生用ともに全画面で統一テンプレート（メニュー部 + 末尾にログアウト等）
- 学校セットアップは `wizardStep(n)` で4ステップ切替(`.wizard-step.done/.active`、`.wizard-pane.active`)
- 学校詳細は `detailTab(tab, key)` で5タブ切替(`.detail-pane.active`)
- 進捗可視化3段: 一覧の `.stepper-mini` / 詳細&先生ダッシュの `.progress-stepper` / 次アクションの `.next-action-banner`
- プロンプトは二面分離(v3.1〜): 正本=構造化カード `.persona-card` / 揮発=授業運営右パネル `.volatile-badge` で区別
- 授業運営は v3.1〜 レイアウト `.classroom-layout-v31` (フェーズバー + 生徒グリッド + フェーズ別中央ビュー + ツールキット)
- 生徒ドリルダウンは `openStudentDetail(name)` で会話履歴+ワークシート+個別メモを表示
- 生徒画面では「アイスブレイク/本題トーク」という内部用語は出さず「あなたの考えた質問/意見」を使う
- v3.5: モックアップ告知バナー `.mock-banner` を全画面固定表示(`body.mock-banner-on` で上部余白)。✕で閉じられる
- v3.5: `location.hash` によるディープリンク対応(`index.html#admin-schools` などでサイトマップから直接遷移可能)
- v3.5: 学校名は全て匿名化(A〜D)。校長名は「例)山田 太郎」のように `例)` を冠する
- v3.6: 生徒サマリー視聴画面 `student-summary`(7-5-G)を追加。先生の投影操作で表示されるビュー — サマリー本文(読取専用)+先生⇄AIの対話(実況風)+デモ用に①/②切替ボタン。JS `setStudentSummary(n)` と `STUDENT_SUMMARIES` 辞書で内容切替
- **v3.7 サマリー方針**: 生徒に投影・共有される可能性のあるサマリー(PHASES_V32 / `#summary-body` / teacher-summary / STUDENT_SUMMARIES)からは以下を排除:
  - 個人名・個別発言の直接引用(「田中さん『〜』」など)→ カテゴリ集約と件数のみ
  - 次フェーズのネタバレ(「次のポジショニングでは〜」「次の本題では〜」など)→ 削除
  - 特定の結論への誘導(「〜を引き出しやすそう」「〜を軸に問うと深まる」など)→ 中立的な「傾向」記述のみ
  - 授業運営画面の先生用ツール(発話ストリーム・トップ発話者・ハイライトカード・先生メモ)は生徒非表示のため対象外
