## 1. 概要

### 1-1. プロジェクト名
**AI校長**(授業名・アプリ名ともに同じ)

### 1-2. プロジェクトの目的
学校の校長先生をモデルにしたAI(または校長代理AI)と児童生徒が対話することで、AI・情報社会について学ぶ授業用Webアプリ。
既存アプリの大幅な機能拡張・UI改修を行う。

### 1-3. 本仕様書のゴール
- 必要な画面の洗い出し
- 画面遷移の明確化
- ユーザーごとの権限・アクセス範囲の整理
- エンジニアが実装に着手するための共通理解の構築
- クリッカブルモックアップ作成のための設計図とする

### 1-4. 既存アプリの現状(認識)
- 生徒は配布されたURLにアクセスし、PTT(Push to Talk)でAIと会話する
- 会話内容はLINEのトーク画面のように画面上に表示される
- 画面上部にAIのアイコンと名前が表示される
- 「押して話す」ボタンと「体験終了」ボタンがある

## 2. 用語定義

| 用語 | 意味 |
|------|------|
| **管理者** | 運営側のスタッフ。すべての学校・授業に対する権限を持つ |
| **学校の先生** | 各学校の校長先生または担当の先生。自校のみ操作可能 |
| **生徒** | 授業に参加する児童生徒 |
| **AI校長モデル** | 学校ごとに作られるAIキャラクター。プロンプト・声・画像などの設定の集合体 |
| **校長本人Ver** | 校長先生自身をモデルにしたAI |
| **校長代理Ver** | 校長先生の代理として位置付けるAI(マスコット等を使用) |
| **授業ルーム** | 1回の授業(1クラス分)を行うための仮想空間。専用URLが発行される |
| **フェーズ** | 授業内の段階(アイスブレイク / 本題トーク / ポジショニング 等) |
| **事前インタビュー** | AI校長モデル作成前に校長/担当の先生が回答するアンケート |
| **事前ワークシート** | 生徒が授業前に行うメモ記入機能(オン/オフ可能) |
| **サマリー** | 授業中の生徒の発言を集約・分析したもの |

## 3. ユーザー階層とデータ構造

### 3-1. 3階層のユーザー構造

```
┌─────────────────────────────────────────┐
│  管理者(運営側)                          │
│  ・全学校にアクセス可能                  │
│  ・先生の機能もすべて使える              │
│  ・AI校長モデルの作成は管理者のみが行う  │
└──────────────────┬──────────────────────┘
                   │ 学校ごとに作成
                   ▼
┌─────────────────────────────────────────┐
│  学校の先生(校長先生 / 担当の先生)       │
│  ・自校のみアクセス可能                  │
│  ・授業の運営を担当                      │
│  ・AI校長モデルの調整は可能(作成は不可)│
└──────────────────┬──────────────────────┘
                   │ 授業ごとに発行
                   ▼
┌─────────────────────────────────────────┐
│  児童生徒                                │
│  ・配布された授業ルームURLからアクセス   │
│  ・ログイン不要                           │
└─────────────────────────────────────────┘
```

### 3-2. データの入れ子構造

```
学校 (School)
  │  ・学校名、所在地、担当の先生のアカウント情報
  │
  └─ AI校長モデル (AIPrincipalModel) ※学校に1つ
       │  ・種別: 校長本人Ver / 校長代理Ver
       │  ・プロフィール、写真、声、プロンプト等
       │  ・事前インタビューの回答内容
       │
       └─ 授業ルーム (ClassRoom) ※クラス数分作成可能
            │  ・専用URL(ランダムトークン付き)
            │  ・有効期間(開始日時・終了日時)
            │  ・授業の事前設定(後述)
            │
            └─ 生徒 (Student) ※参加人数分
                 ・末尾IDのみ異なる個別URL
                 ・名前
                 ・(オプション)事前ワークシートの記入内容
```

### 3-3. 全体のデータ規模イメージ

- 1校 = 1つのAI校長モデル
- 1つのAI校長モデル = 複数の授業ルーム(クラス数分)
- 1つの授業ルーム = 複数の生徒(クラスの人数分)

例:A中学校(3クラス、各30人)で実施 → モデル1個・授業ルーム3個・生徒90人分のセッション

## 4. AI校長モデルの2種別

AI校長モデルは作成時に以下のどちらかを選択する。

### 4-1. 校長本人Ver
校長先生自身をモデルにしたAI。校長のアバターとして位置付ける。

**必要な要素**
- 校長先生の顔写真(撮影またはアップロード)
- 校長先生の声(録音)
- プロフィール(校長先生自身、学校、地域に関すること)
- 雰囲気・口癖・方言などのテイスト設定

### 4-2. 校長代理Ver
校長先生の代理として位置付けるAI。

**必要な要素**
- アイコン画像(校章、マスコットなど)
- デフォルトの合成音声(録音不要)
- プロフィール(学校、地域に関すること、マスコットキャラの設定など)
- 雰囲気・口癖・方言などのテイスト設定

### 4-3. 共通の簡易化方針
事前インタビューの負担を軽くするため、以下を行う。
- 学校便りなどの公開情報から、あらかじめ下書きプロンプトを作成しておく
- 一問一答のアンケート形式
- 選択式のアンケートも併用

## 6. アプリ構成と画面分類

### 6-1. 1つのWebアプリで構成
本システムは **1つのWebアプリ** として構築し、ログイン後の役割によって表示される機能が変わる方式とする(RBAC = Role-Based Access Control)。
これにより、開発・保守コストを抑え、「先生ができることを管理者が全部できる」という要件を自然に実現する。

### 6-2. 画面の分類

| 分類 | 対象ユーザー | アクセス方法 |
|------|-------------|-------------|
| **管理者画面** | 管理者 | ログイン |
| **先生画面** | 学校の先生・管理者 | ログイン |
| **事前インタビュー画面** | 校長先生 / 担当の先生 | 専用URL(ログイン不要) |
| **生徒画面** | 生徒 | 専用URL(ログイン不要) |

### 6-3. URL設計の考え方

| URL種別 | 形式の例 | アクセス制御 |
|---------|---------|-------------|
| 管理画面 | `/admin/...` | ログイン必須 |
| 事前インタビュー | `/interview/{ランダムトークン}` | トークン保有者のみ。有効期限あり |
| 授業ルーム(生徒) | `/class/{ランダムトークン}` | トークン保有者のみ。授業時間内のみ有効 |
| 生徒個別URL | `/class/{トークン}/s/{生徒ID}` | 端末識別のためルームトークン+末尾ID |

トークンは推測困難なランダム文字列(例:16文字以上の英数字)。

## 7. 画面詳細

### 7-1. 共通画面

#### 7-1-A. ログイン画面
- メールアドレス・パスワード入力
- ログインボタン → ロールに応じた画面へ遷移
  - 管理者 → 学校一覧画面(7-2-A)
  - 学校の先生 → 自校のダッシュボード(7-3-A)
- パスワードリセットリンク

---

### 7-2. 管理者画面

#### 7-2-A. 学校一覧画面
- 登録されている全学校のリスト表示
  - 学校名、所在地、担当の先生、AI校長モデル状態、最終授業日 等
- 「新規学校を追加」ボタン → 7-2-B
- 各学校行をクリック → その学校のダッシュボード(7-3-A)へ遷移

#### 7-2-B. 学校追加・編集画面
- 学校名、所在地
- 担当の先生のアカウント作成(メールアドレス、初期パスワード等)
- 保存 → 学校一覧へ戻る

#### 7-2-C. AI校長モデル作成画面 ※管理者専用
- 対象学校を選択
- 種別を選択(校長本人Ver / 校長代理Ver)
- 事前インタビューの回答内容を取り込んでプロンプトを生成
- 生成されたプロンプトを編集
- 保存 → AI校長モデルが完成

> 管理者は、各学校のダッシュボードに入った後、先生と同じ画面・機能をすべて使用できる。
> 学校選択 → 「先生としての画面」に切り替わるイメージ。

---

### 7-3. 先生画面(管理者も使用)

#### 7-3-A. 学校ダッシュボード
画面構成:左側にサイドメニュー、右側にメインコンテンツ。

**サイドメニュー項目**
- ホーム(本画面)
- AI校長モデル管理 → 7-3-B
- 事前インタビュー管理 → 7-3-D(管理者のみ表示される項目あり)
- 授業ルーム一覧 → 7-3-E
- (管理者のみ)学校一覧へ戻る

**メインコンテンツ**
- AI校長モデルの状態(未作成 / 作成済み)
- 最近の授業ルーム
- 直近の予定
- クイックアクション(新規授業ルーム作成 等)

#### 7-3-B. AI校長モデル詳細・編集画面
- 種別表示(本人Ver / 代理Ver)
- アイコン画像、声サンプル(本人Verのみ)、プロフィール表示
- **プロンプト編集エリア**(口調・雰囲気の微調整)
  - 編集後「保存」「テストへ」ボタン
- 「会話テストへ」ボタン → 7-3-C

> モデルの新規作成・削除は管理者のみ。先生は閲覧と微調整のみ。

#### 7-3-C. 会話テスト(セルフチェック)画面
- 先生自身がAI校長と会話してハルシネーションや口調を確認できる
- 生徒画面と同じ会話UI
- 横にプロンプト編集パネル(編集 → 即反映)
- テスト履歴の閲覧

#### 7-3-D. 事前インタビュー管理画面
- 自校の事前インタビューの状態(未送付 / 回答中 / 回答完了)
- 回答内容のプレビュー
- (管理者のみ)「インタビューURLを新規発行」ボタン → トークン付きURLが生成される
- (管理者のみ)発行したURLのコピー・有効期限管理

#### 7-3-E. 授業ルーム一覧画面
- 自校の授業ルームのリスト
- 各ルームの状態(予定 / 進行中 / 終了)、開始日時、参加生徒数
- 「新規授業ルームを作成」ボタン → 7-3-F
- 各ルーム行をクリック → 授業設定画面(7-3-F)または運営画面(7-3-G)

#### 7-3-F. 授業ルーム作成・設定画面
**基本情報**
- 授業ルーム名(例:「3年A組 11月20日」)
- 開始予定日時 / 終了予定日時(URLの有効期間に直結)

**フェーズ構成設定**
授業中に進行する各フェーズの有効/無効・順序を設定。
- ☑ アイスブレイク
- ☑ 本題トーク
- ☐ ポジショニング(オン/オフ)
- ☐ 事前ワークシート(オン/オフ)

**テーマ設定**
- アイスブレイクのテーマ(例:「校長先生に学校・地域のことを質問してみてください」)
- 本題トークのテーマ(デフォルト or カスタム入力)
  - 例:「学校や地域の良さは?」「変えたい校則はある?」
- ポジショニングのテーマ(両端のラベル設定。例:左「校長先生と言える」⇔ 右「言えない」)

**生徒用URLの発行**
- 「URLを発行する」ボタン → 7-3-Fの下部に表示される
- URLのコピー、QRコード表示
- 事前ワークシートが「オン」の場合、事前ワークシート用URLも発行される

#### 7-3-G. 授業運営画面(授業中のメイン画面)
**画面構成(3ペイン想定)**
- 上部:現在のフェーズ、進行ボタン(「次のフェーズへ」「現在のフェーズを終了」)
- 左:全生徒のリスト・参加状況・ポジショニング表示エリア
- 中央:全生徒の会話のリアルタイム表示(発言一覧)
- 右:プロンプト調整パネル / サマリーパネル(切替)

**操作機能**
- フェーズ切り替え(全生徒の画面が一斉に同期)
  - 例:「アイスブレイクを開始」「アイスブレイクを終了」「本題トークへ」「ポジショニングへ」
- 個別生徒の発言の確認
- リアルタイムでプロンプト調整(口調などを変えるとすぐ反映)
- サマリーパネルを開く → 7-3-H

#### 7-3-H. サマリー画面
- フェーズごとにサマリーを生成
  - アイスブレイクのサマリー / 本題トークのサマリー
- **対話型インターフェース**でサマリーの整理が可能
  - 例:「多かった意見順に並べて」「賛成・反対の比率を教えて」「理由とともに3つに整理して」
- サマリーをエクスポート(コピー、PDFダウンロード等)

#### 7-3-I. 授業終了後の振り返り画面
- 授業ルームの会話履歴の閲覧
- 最終サマリーの閲覧・出力
- 校長先生・学校としてのフィードバック記録(任意)

### 7-4. 事前インタビュー画面(URL/ログイン不要)

校長先生または担当の先生が、管理者から送られたURLを開いてアクセスする。

#### 7-4-A. インタビュー開始画面
- 「校長AIを作成するためのインタビューです」等の説明
- 種別の自動表示(本人Ver / 代理Ver)※URLにより決定済み
- 所要時間の目安
- 「開始する」ボタン → 7-4-B

#### 7-4-B. アンケート画面(ステップ式)
複数ステップに分かれた一問一答形式。
- 学校・地域に関する選択式 + 記述式の質問
- 本人Verの場合は校長個人に関する質問も含む
- ステップごとに「次へ」「戻る」
- 進捗バー表示

#### 7-4-C. 声の録音画面 ※本人Verのみ
- 指定の文章を読み上げて録音
- 再生して確認 → 録り直し or 次へ

#### 7-4-D. 写真の撮影・アップロード画面
- 本人Ver:校長先生の顔写真
- 代理Ver:校章 / マスコット等のアイコン画像
- カメラ撮影 or ファイルアップロード

#### 7-4-E. テイスト設定画面
- 雰囲気(やさしさ、フォーマル度などの選択)
- 口癖、方言の入力
- 自由記述

#### 7-4-F. 完了画面
- 「ご回答ありがとうございました」
- (任意)回答内容の確認
- 以降、URLは無効化または閲覧専用化

---

### 7-5. 生徒画面(URL/ログイン不要)

生徒は、配布された授業ルームURLにアクセスする。
URLは末尾にIDが自動付与される(例:`/class/{ルームトークン}/s/abc123`)。
端末ごとに異なるIDが発行され、同じ生徒は同じIDで識別される。

#### 7-5-A. 名前入力画面(初回アクセス時)
- 「名前を入力してください」
- 入力フォーム + 「参加する」ボタン
- 入力後 → 待機画面(7-5-B)

#### 7-5-B. 待機画面
- 「先生の指示を待っています」
- 自分の名前と参加状況を表示
- 先生がフェーズを開始すると自動で次画面へ遷移

#### 7-5-C. 会話画面(アイスブレイク / 本題トーク 共通)
**画面構成**(既存アプリの構成を踏襲)
- 上部:AI校長のアイコン・名前(「詳細を表示」リンク)
- 中央:LINE風のチャット履歴
- 下部:大きな「押して話す」ボタン(PTT)
- 右上:「体験終了」ボタン

**追加要素**
- フェーズ表示(現在「アイスブレイク」または「本題トーク」)
- 現在のテーマ表示(画面上部)
- ワークシート表示(オン時のみ。事前記入したメモがカンペとして表示)
- 先生がフェーズを切り替えると、画面が自動で切り替わる(同期)
- 先生が停止操作を行うと、会話が止まる

#### 7-5-D. ポジショニング画面(オン時のみ)
- 画面中央に直線(ポジショニングマップ)
- 両端にラベル表示(例:左「校長先生と言える」⇔ 右「言えない」)
- 自分のマーカー(名前付き)をドラッグで移動
- 他の生徒のマーカー(名前付き)もリアルタイム表示
- いつでも位置を変えられる

#### 7-5-E. 終了画面
- 「お疲れさまでした」
- (任意)感想入力欄

#### 7-5-F. 事前ワークシート記入画面(オン時のみ・授業前)
- 別URLでアクセス(または同URLで授業時間外にアクセスした場合に表示)
- フェーズごとの問い(アイスブレイク用 / 本題トーク用)に対する事前メモを入力
- 名前入力 + 記入内容
- 端末にクッキーで識別IDを保存し、当日の授業ルームURLにアクセスした際に紐付ける
- **フォールバック**:クッキーが取れない場合(別端末・シークレットモード等)は、当日に名前で検索して紐付け、または当日改めて入力

## 9. 主要機能の詳細仕様

### 9-1. フェーズ同期(全ユーザー一括管理)
- 先生が授業運営画面でフェーズ切替操作を行うと、その授業ルームに参加している全生徒の画面が一斉に切り替わる
- 同じ会話の中でフェーズを分けて運用できる(会話履歴は保持される)
- フェーズの種別:
  - 待機
  - アイスブレイク(会話)
  - サマリー表示(生徒側は待機 or 簡易表示)
  - 本題トーク(会話)
  - ポジショニング
  - 終了

### 9-2. ポジショニングのリアルタイム共有
- 生徒画面・先生画面の両方で全員分のマーカーがリアルタイムで見える
- 各マーカーには名前を表示
- 自分のマーカーは色や枠で強調
- 移動はいつでも・何度でも可能

### 9-3. 事前ワークシート(クッキーによる識別)
- オン/オフを授業ルーム単位で設定可能
- 事前ワークシート用URLは授業ルームURLとは別(または同URLで授業時間外アクセス時に切替表示)
- クッキーに識別IDを保存し、当日の授業ルームURLにアクセスした際に同じIDなら自動紐付け
- フェーズごとに、事前記入したメモが会話画面のカンペとして表示される
- **エンジニア相談ポイント**:クッキーが取れない場合のフォールバック方法(名前で検索/当日再入力)

### 9-4. サマリー出力(対話型・改良)
- フェーズごとにサマリーを生成可能
- 生成にもたつかない(レスポンスを早くする)
- サマリーの整理方法をその場でプロンプト指示できる対話型UI
  - 例:「多かった意見順に並べて、理由とともに」
  - 例:「賛成・反対の比率を教えて」
  - 例:「キーワードを抽出して」
- サマリーのエクスポート機能

### 9-5. プロンプト調整(改良)
- UIをわかりやすく(現状の課題)
- プロンプトを変更すると同じ会話の中でもすぐに反映される
- プリセット(やさしい / フォーマル / 方言強め 等)からの選択も可能
- 変更履歴の保存

### 9-6. 限定URL(時間制限・トークン)
- 授業ルームURLは推測困難なトークンを含む
- URLを知っている人しかアクセスできない
- その授業の時間しかアクセスできない(開始時刻〜終了時刻)
- 期限切れ後は「この授業は終了しました」表示

### 9-7. 校長AIセルフチェック機能
- 校長先生または担当の先生が、授業前にAI校長と実際に会話して挙動を確認できる機能
- 会話テスト画面(7-3-C)に該当
- 横でプロンプトを編集しながら何度でも試せる

### 9-8. 授業の事前設定(管理者・先生)
- ポジショニングをやる/やらないの選択
- 事前ワークシートのオン/オフ
- アイスブレイクのテーマ入力
- 本題トークのテーマ入力(デフォルト or カスタム)
- ポジショニングの両端ラベル設定

## 11. モックアップで省略する範囲

クリッカブルモックアップでは、以下を省略する。

| 項目 | モックアップでの扱い |
|------|---------------------|
| 実際のAI応答 | ダミーテキストで代替(例:「これは校長先生のサンプル回答です」) |
| 音声録音・再生 | UIだけ表示、実機能なし(録音ボタンを押すと「録音完了」表示するだけ) |
| 合成音声出力 | なし(テキスト表示のみ) |
| リアルタイム同期 | 画面遷移のシミュレーションのみ(複数端末同期の実装はなし) |
| 認証・ログイン処理 | フォームに何を入れてもログインボタンで次画面へ |
| データの永続化 | ページリロードでリセットされてOK |
| URLトークンの生成 | ダミーの固定文字列でOK |
| クッキーによる生徒識別 | 表示の確認のみ(機能としては動かさなくてOK) |

## 13. 参考:授業の50分構成

> 授業の流れの参考。アプリ仕様の根拠となる元情報。

### 13-1. 授業の説明
- 情報社会について
- AIについて知っていることの発表

### 13-2. 校長AIの体験:アイスブレイク
- スタッフが前で使い方を説明する
  - 話したことは校長先生に伝わる
  - 校長先生のアバターor代理の存在である
- 生徒が配布された一時的なURLにアクセス
- 各自ヘッドセットを使用して会話する
- 「校長AIに質問してみてください」
  - 本人Ver:校長先生と学校・地域のことについて質問
  - 代理Ver:学校・地域のことについて質問

### 13-3. 感想を共有
- どんな返答があったか?(事実 / それっぽい / 嘘=ハルシネーション)
- (任意)校長AIはどうだった?
  - 本人Ver:校長先生ぽかった?
  - 代理Ver:どんな存在?
- (任意)校長AIの喋り方はどうだった?どんな風だと喋りやすい?

### 13-4. (任意)校長登場・サマリー表示
- 校長が登場し、みんなの意見が一覧で見れることを伝える
- 一気に話しかけられる、いろんな人と同時に会話できて情報が集まる
- 管理者画面やサマリーを表示する

### 13-5. (任意)プロンプト調整
- 生徒の意見を反映し、どんな風に返答してほしいか(優しさ、口調など)をその場でプロンプト変更して反映

### 13-6. 校長AIの体験:話し合い
- 導入:校長先生からみんなに聞きたいこと、話し合いたいことがあります
- デフォルトテーマ:学校や地域の良さ、学校・地域の名物
- 任意テーマ:実際に校長先生が話したいテーマ(例:変えたい校則は?)
- 授業後にフィードバックを返せるテーマが好ましい

### 13-7. 体験の振り返り:サマリー表示
- 校長先生または担当の先生が校長AIシステムのサマリーを確認
- たくさんの意見が出たことと、出た意見のいくつかを振り返る
- サマリーに対してプロンプトを入力し、整理して確認
- 校長先生がコメントする、または担当の先生が「校長先生にも伝えてみてもらいましょう」という

### 13-8. ディスカッション・ポジショニング

**方法1**
- テーマ:校長AIは校長先生(または校長代理)と言えるのか?
- ポジショニングマップを使う
- 周りの人とも話し合い、随時マッピングを更新する

**方法2**
- トークテーマがいくつか用意されている
- テーマごとにグループに分かれて話し合う
- 最後にグループで話したことを発表する

### 13-9. 校長AIの仕組み紹介・AIレクチャー
- 複数のAIが組み合わさっている
- 生成AIの仕組み(確率で組み合わせている)
- ネガティブ:ハルシネーション
- ポジティブ:セレンディピティ

### 13-10. アクロバティック案(発展)
- 授業内で事前インタビューを行う(特に声の収録など)
- 質問攻めしてハルシネーションを引き出す
- 実際に授業後に影響のあるテーマをトークテーマに設定する
